## Similar Implementations

### Prior WS-server spec

- **Location:** `agent-os/specs/2026-05-09-1700-ws-sync-server/`
- **Relevance:** Established the daemon architecture this spec extends. Defines `RoomRegistry`, `RoomRuntime`, `ClientConnection`, the `IAppConfig` keys (`ws_host`, `ws_port`, …), and the `occ playbacksync:ws-serve` entry point. The new admin HTTP server hangs off the same React event loop already running in `WsServe`.
- **Key patterns:** in-memory-only state model, `IAppConfig` for daemon configuration, no DB schema changes for ephemeral state, graceful-degradation responses.

### `docs/ws-protocol.md`

- **Location:** `docs/ws-protocol.md`
- **Relevance:** Source of truth for the WS protocol surface. Confirms `clientId` is daemon-generated and returned to clients in `ROOM_STATE`, that JOIN authenticates against the room password (no NC user identity), and that no `MEMBER_JOINED`/`LEFT` broadcasts exist (presence is server-only). The new "Admin HTTP" section appends here.
- **Key patterns:** documented config keys (extend with `ws_admin_host`/`ws_admin_port`/`ws_admin_secret`), error-code table style, JSON envelope shape.

- **Location:** External — `nextcloud-spreed-signaling` project, "backend HTTP API" docs.
