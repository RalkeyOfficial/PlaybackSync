# Missing Features Report ‚Äî OLD vs CURRENT (backend + dashboard only)

Comparison of features documented in `OLD_CODE/docs/` against the current `docs/` and `agent-os/specs/`. Browser-extension features are excluded ‚Äî backend (PHP / Nextcloud app, WebSocket sync daemon) and dashboard/admin panel only.

## Truly absent (no Phase 1/2 mention)

- **Prometheus metrics** (`/metrics`, `playbacksync_*` gauges/counters). OLD ¬ß10. Absent from current docs and roadmap.
- **Global per-room broadcast flood control.** OLD ¬ß8 had both per-connection *and* global rate limits ‚Äî current `docs/ws-sync-server.md` only mentions per-connection (`ws_rate_limit_events_per_sec`).
- **Graceful shutdown / `SERVER_SHUTDOWN` message.** OLD broadcast a notice on SIGTERM. Current accepts "daemon restart drops connections, clients reconnect" as a design tradeoff ‚Äî but no graceful-close path is specified.
- **WebSocket test harness** (`scripts/test-harness.js` from OLD ¬ß14). Phase 1 has PHPUnit but no multi-client WS simulator.
- **Resource budget estimates** (CPU/RAM/disk per room). OLD ¬ß15.

## Deferred per current roadmap (not gaps, just FYI)

- **`last_state` DB column** (cached playback state) ‚Äî Phase 2 migration noted in `docs/architecture.md`.
- **Token-based auth** (replace password-in-URL). OLD ¬ß17 future work.
- **Live playback controls in dashboard** ‚Äî OLD itself marked this out of v1.
- **Event log surfaced in UI** ‚Äî exists server-side as in-memory ring buffer, no API exposure yet.

## Intentionally dropped/redesigned (worth confirming)

- **`COMMAND` message type** ‚Üí folded into STATE.
- **`TIME_REPORT`** ‚Üí replaced by HEARTBEAT carrying `currentPos`.
- **`MEMBER_JOINED`/`MEMBER_LEFT` broadcasts** ‚Äî current `docs/ws-sync-server.md` explicitly says no client-facing presence.
- **Public ACK/NACK protocol** ‚Äî kept as internal server concept only.
- **Standalone Node.js Docker service** ‚Üí replaced by `occ playbacksync:ws-serve` PHP daemon.

## Recommendations to address

If you want parity with the OLD design's operational story, the highest-value items to add to the roadmap are:

1. **Graceful shutdown semantics** ‚Äî at minimum document the contract (e.g. clients receive close code X and reconnect with backoff).
2. **Metrics story** ‚Äî even if not Prometheus, document what's observable.

Resolved since this file was first written:

- **Kick endpoint** ‚Äî `DELETE /api/v1/rooms/{uuid}/clients/{clientId}` is now wired end-to-end (PHP controller ‚Üí HMAC loopback ‚Üí daemon `KickController`) with a per-client reconnect block (`ws_kick_block_ms`, default 30 s) and a `KICKED` error frame on the way out. See `agent-os/specs/2026-05-10-1415-connected-client-kick/`.
- **Healthcheck endpoint** ‚Äî daemon now exposes unauthenticated `GET /healthz` on the loopback admin port (`ws_admin_port`, default 8766) returning aggregate counters (active rooms, connected clients, uptime, tick freshness) plus daemon version. Single-path carve-out evaluated before the HMAC check; response carries no UUIDs / client IDs / secrets. A public `GET /api/v1/health` (`#[PublicPage]`) loopback-passthrough route surfaces the same body wrapped with a reachability envelope so external probes (k8s, status pages) can hit a stable Nextcloud URL ‚Äî always returns HTTP 200, daemon trouble surfaces as `status: "degraded"`. Bonus: `/api/v1/ws/status` was refactored to actually probe the daemon (it previously only checked install-state) and now distinguishes `not_installed` from `not_running` so the dashboard surfaces a warning state with restart guidance instead of misleadingly reporting "available". See `agent-os/specs/2026-05-10-1530-ws-daemon-healthcheck/`.
- **Participant list / live presence in API** ‚Äî rooms responses now carry a `live` block sourced from the WS daemon over a loopback admin port (default `127.0.0.1:8766`) signed with an HMAC-SHA256 shared secret over `(method, path, ts)`, ¬±30 s replay window. PHP enriches `GET /api/v1/rooms` and `GET /api/v1/rooms/{uuid}` with anonymous `clientId` presence, live `playerState`/`videoPos`, content identity (`providerId`, `episodeId`, `pageUrl`, `contentKey`), and `lastActivityMs`; per-room client list capped at 50 with `connectedCount` reflecting the true total. Daemon-unreachable / HMAC-mismatch / timeout paths collapse to `live: null` per room ‚Äî rooms list still renders. Shared secret is auto-generated by an `IRepairStep` on `app:enable` and every `occ upgrade` and stored sensitive in `IAppConfig`; operators never touch `openssl rand`. No DB writes for ephemeral state. `RoomCard.vue` gained click-to-open detail modal that refetches on open, plus a footer viewers count. See `agent-os/specs/2026-05-09-1900-rooms-api-live-state/`.
- **Public share endpoint** ‚Äî `GET /apps/playbacksync/r/{uuid}` is now wired end-to-end. Public route (`#[PublicPage]`, `#[NoCSRFRequired]`) gates on HTTP Basic Auth: the username is ignored, the password is verified against the room's argon2id hash via `RoomService::verifyPassword`, and on success the visitor is 302-redirected to `room.targetUrl` with `sync_url=wss://<host>/apps/playbacksync/ws/{uuid}` and `sync_password=<plaintext>` merged into the query (existing query params and fragments preserved). Failed attempts feed Nextcloud's `IThrottler` under action `playbacksync_share`; missing/malformed `Authorization` headers prompt the browser without registering as attempts. Unknown and expired rooms collapse to the same `404 {"error":"not_found"}` surface ‚Äî never leaks "expired". Contract mirrors OLD_CODE Fastify route. See `agent-os/specs/2026-05-10-1615-public-share-endpoint/`.
